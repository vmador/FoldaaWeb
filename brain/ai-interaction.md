# Brain & AI Interaction Manifesto

The **Brain** is the cognitive core of every **Tent**. It ensures that no context is lost, no decision is undocumented, and the transition between Builders is seamless.

## 1. Source of Truth (SoT) Analysis
The Brain continuously monitors the 4 core manifest files (`system.md`, `design.md`, `roadmap.md`, `stack.md`).
- **Audit**: It checks for inconsistencies between the design and the system implementation.
- **Suggestions**: It proposes enhancements or checklists based on the roadmap milestones.

## 2. Decision History (The Memory)
The Brain maintains a chronological log of "Why X was done".
- **Tracking**: Every significant architectural or design change is recorded with its rationale.
- **Recall**: When a new **Builder** enters the Tent, the Brain summarizes the history to provide immediate context.

## 3. Automated Reviews & Quality
- **Review Loop**: The Brain analyzes PRs and code commits against the **Source of Truth**.
- **Standards**: It ensures that the code matches the specified **Stack** and **Design** guidelines.
- **Feedback**: It provides automatic review comments to Builders before final human approval.

## 4. Transferability Management
When the capacity of a Tent changes (e.g., a Builder leaves), the Brain:
- **Offboarding**: Captures any last-minute context or undocumented "flames".
- **Onboarding**: Greets the new Builder with a "Context Injection" - a summary of the current state, active tasks, and recent decisions.

## 5. Learning & Feedback Loop
The Brain learns from each **Temp**.
- **Optimization**: If a specific module configuration works well in one Temp, the Brain suggests it as a standard for future similar projects.
- **Analytics**: It tracks Builder velocity and module completion rates to improve ROI predictions for the **Igniter**.
